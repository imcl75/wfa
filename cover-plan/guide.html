Release Schedule — Staff Guide

How to view and manage the release schedule

Part 1  ·  Viewing the schedule

The release schedule is a live web page — it updates automatically and can be viewed on any device including phones, tablets and laptops. You do not need to log in or install anything.

How to access it

1
Open the schedule in your browser

Go to the following address:

https://imcl75.github.io/wfa/cover-plan/

Bookmark this page so you can find it easily next time.

The schedule refreshes automatically every 3 minutes. You can also tap refresh now in the top bar at any time to get the latest version immediately.

The three views

Week view

Shows the full week as a grid — one column per day, one row per session (Before School, AM, Lunch, PM, After School). Today's column is highlighted in blue. Use the arrow buttons to move between weeks. Cards are colour-coded by year group.

Today view

Shows only today's arrangements as a flat list, grouped by session. Useful for a quick check at the start of the day.

My view

Shows your personal schedule — every session where you are out of class or covering someone else, grouped by week. Select your name from the dropdown the first time you use it; the page will remember your choice.

Understanding the cards

Each card shows:

  • Name of the teacher who is out
  • Cover Staff — who is covering them
  • Reason badge (e.g. PPA, ECT1, Training)
  • Time badge — if the session is only part of a session (e.g. 10:00 – 11:30)
  • Notes — any additional information

Whole-school events (such as Staff Meetings, FLC, or PAC) appear as a full-width banner with an All Staff label rather than individual cards.

Long-term absences

If a member of staff is on long-term sick leave, an orange banner will appear at the top of every view showing their name, role, and any relevant notes (e.g. expected return date).

For the cover manager

Part 2  ·  Managing the schedule

The schedule is driven by a Google Sheet. When you update the sheet, the viewer picks up the changes automatically within 3 minutes. You can also notify all staff immediately using the Release Schedule → Publish & Notify Staff menu in the sheet.

Setting up the term schedule

Instead of entering every recurring arrangement by hand each term, use the Recurring tab and the Generate function to fill the Plan tab automatically.

The Recurring tab uses the same columns as Plan but without dates — just the repeating pattern:

ColumnNotes
DayMust be exactly: Monday Tuesday Wednesday Thursday Friday
SessionSame values as Plan tab. Leave blank for All Day.
Teacher OutFirst name, as in the Plan tab.
Cover StaffWho regularly covers this slot.
Reasone.g. PPA, ECT1.
NotesOptional.
Time From / Time ToOptional, if the session is a fixed time.
1
Fill in the Recurring tab

Add one row for each arrangement that repeats every week — e.g. Innes out on Monday AM for PPA. You only need to do this once; update it if the pattern changes permanently.

2
At the start of each term: Release Schedule → Generate Term Schedule

Enter the term number when prompted. The script reads the Recurring tab and the Weeks tab, calculates the correct date for each arrangement in each teaching week, and adds all the rows to the Plan tab automatically. Entries that already exist are skipped.

3
Edit exceptions in Plan as the term progresses

If an arrangement changes for a specific day, just edit or delete that row in Plan directly. The Recurring tab does not need updating for one-off changes.

If the pattern changes from a certain week onwards — for example, Innes's PPA moves from Monday to Tuesday from week 3 — update the Recurring tab first, then use Release Schedule → Regenerate from Week… and enter the week label (e.g. T5W3). The script will replace all auto-generated entries from that week onwards with the updated pattern, while leaving any manually added entries intact.

Notifying staff

1
Finish updating the Plan tab

Make sure all cover arrangements for the day are entered and correct.

2
Click Release Schedule → Publish & Notify Staff

This sends an email to all staff with a summary of today's arrangements and a link to the viewer. The viewer updates automatically every 3 minutes, so you only need to send a notification when you want to alert staff to a change.

Staff do not need to do anything to receive the notification — they just need to be on the mailing list in the Apps Script. To update the list, ask your administrator to edit the NOTIFICATION_EMAILS section of the script.
